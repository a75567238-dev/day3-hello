# AI 코드 어시스턴트 교육 프로그램 판매 랜딩페이지 설계

## 1. 목표
- 정적 한 페이지 랜딩으로 40대 이상 성인 잠재고객을 카카오톡 상담으로 전환한다.
- 1차 전환 KPI는 카카오톡 채널 클릭(문의 시작)이다.

## 2. 대상 사용자
- 40대 이상 성인
- 기술 용어보다 결과와 신뢰를 중시하는 사용자
- 학습 난이도/시간 부담에 대한 불안이 있는 사용자

## 3. 톤앤매너
- 신뢰형(차분, 가독성 최우선)
- 과장 표현보다 명확한 가치 전달 우선
- 큰 대비, 넉넉한 여백, 짧은 문장 중심 카피

## 4. 정보구조(한 페이지)
1) 히어로(핵심 가치 + 1차 CTA)
2) 프로그램 효과 3포인트
3) 진행 방식 3단계
4) 신뢰 근거(강사/운영/후기 요약)
5) FAQ
6) 하단 CTA

## 5. 섹션 상세 설계

### 5.1 히어로
- 메인 카피: 비개발자도 이해하는 실전 AI 코드 어시스턴트
- 서브 카피: 40대 학습자도 따라올 수 있는 속도로, 실제 업무에 바로 적용 가능한 방식 강조
- CTA 버튼: 카카오톡으로 상담받기
- 보조 신뢰 문구: 복잡한 이론보다 실습 중심

### 5.2 프로그램 효과 3포인트
- 시간 단축: 반복 작업을 AI로 줄여 업무 효율 향상
- 실무 적용: 배운 내용을 바로 업무에 연결
- 부담 완화: 어려운 개념을 쉬운 언어로 단계별 학습

### 5.3 진행 방식 3단계
1) 카카오톡 상담 신청
2) 현재 수준 진단 및 과정 안내
3) 수강 시작 및 적용 피드백

### 5.4 신뢰 근거
- 강사 소개 요약(실무 경험 중심)
- 운영 방식(질문 응답/실습 비중 등)
- 후기 요약(짧은 인용문 1~2개)

### 5.5 FAQ
- 비개발자인데 가능한가요?
- 어느 정도 시간이 필요한가요?
- 준비물은 무엇인가요?
- 수강 전 상담이 가능한가요?

### 5.6 하단 CTA
- 문구: 카카오톡으로 상담받기
- 보조 문구: 지금 수준에서 시작 가능한 맞춤 안내 제공

## 6. 행동 흐름/데이터 흐름
- 히어로, 중간, 하단의 CTA는 모두 동일한 카카오톡 채널 링크로 연결
- 사용자는 정보 확인 후 CTA 클릭으로 외부 채널(카카오톡)로 전환
- 폼 입력 단계를 제거해 전환 마찰 최소화

## 7. 예외/오류 처리
- 카카오톡 링크 실패 시 대체 연락 수단(전화/이메일 텍스트)을 CTA 하단에 노출
- 모바일에서 하단 고정 CTA 버튼을 제공해 접근성 강화

## 8. 구현 범위(이번 단계)
- 단일 `index.html` 정적 페이지
- 섹션 앵커 스크롤
- 반응형(모바일 우선)
- CTA 링크 연결 구조 포함

## 9. 검증 기준
- 모바일/데스크톱에서 레이아웃 깨짐 없음
- 페이지 내 모든 CTA 클릭 시 동일한 카카오톡 링크로 이동
- 핵심 카피 가독성(대비/크기/줄간격) 확보
- FAQ를 읽고 CTA로 이동하는 흐름이 자연스러움

## 10. 제외 범위
- 결제 시스템 연동
- 회원가입/로그인
- 백엔드 데이터 저장
- 다중 페이지 확장
